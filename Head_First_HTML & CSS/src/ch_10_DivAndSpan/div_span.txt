<div>元素允许你将页面划分为逻辑区或逻辑分组。
逻辑区：页面上彼此相关的一组元素。

例子：给cats和dogs分组

<html>

......

<div id="cats">
	<h2>
		<p>
			/*...concrete content about cats...*/
	   	</p>
	</h2>
</div>

<div id="dogs">
	<h2>
		<p>
			/*...concrete content about dogs...*/
	   	</p>
	</h2>
</div>

......	

</html>

使用<div>元素后，则可以对其中包含的一组元素指定样式了。
#cats {
	background-images: url("leopard.jpg");
}
#dogs {
	background-images: url("mut.jpg");
}

同时可以在<div>元素中嵌套<div>元素。
例如将“cats”和“dogs”的<div>放在一个“pets”<div>中。
<div id="pets">
	<div id="cats">
		....
	</div>
	
	<div id="dogs">
		....
	</div>
</div>
======================================================================================
<div>元素只是纯粹的结构，他在页面中没有“外观”或默认样式。
可以为<div>元素指定一个class。
实际上我们通常把<div>元素称为容器。
在页面中使用<div>，但不要滥用。如果这样做有助于你将页面分解为逻辑区，从而保证结构清晰并便于指定样式，
那么可以增加更多的结构。如果只是为了在页面中创建大量结构而增加<div>，就只会让页面复杂化，而没有任何
好处。
======================================================================================
子孙选择器：
#[div id] [element] {
	
}

例子：
#elixirs h2 {} 选择id为elixirs的所有<h2>元素，无论这个<h2>元素嵌套的有多深。
#elixirs>h2 {} 只有当<h2>是一个id为“elixirs”的元素的直接孩子时才会被选择。
#elixirs blockquote h2 {} 这会选择<blockquote>下一级的<h2>元素，而且这个<blockquote>是一个id为“elixirs”元素的子元素。
======================================================================================
<span>元素：
<div>元素允许你为块级内容创建逻辑划分，<span>元素则采用类似的方式建立内联内容的逻辑分组。
